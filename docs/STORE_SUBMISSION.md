# Microsoft Store Submission Guide

This guide walks you through building the `.msix` package and submitting it to the **Microsoft Partner Center**.

---

## Prerequisites

- **Windows 10 or 11** (MSIX packaging requires Windows)
- **Node.js ≥ 18** and **npm ≥ 9**
- A **Microsoft Developer Account** — [register at Partner Center](https://partner.microsoft.com/dashboard) ($19 one-time fee)
- **GEMINI_API_KEY** set in `.env.local` (optional but recommended)

---

## Step 1 — Reserve Your App in Partner Center

1. Sign in to [Microsoft Partner Center](https://partner.microsoft.com/dashboard).
2. Go to **Windows & Xbox → Overview → + New product → App**.
3. Enter a unique app name (e.g. **"UWP Weather Forecast"**) and click **Reserve product name**.
4. From the App identity page, copy:
   - **Package/Identity/Name** (e.g. `12345YourName.UWPWeatherForecast`)
   - **Package/Identity/Publisher** (e.g. `CN=XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX`)
   - **Publisher display name**

---

## Step 2 — Update the Electron Builder Config

Open `electron-builder.yml` and set your real values:

```yaml
appId: "12345YourName.UWPWeatherForecast"   # from Partner Center
win:
  publisherName: "Your Publisher Display Name"

msix:
  identityName: "12345YourName.UWPWeatherForecast"
  publisher: "CN=XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX"
  publisherDisplayName: "Your Publisher Display Name"
```

Also update the in-app **Store Packager** form (the "Microsoft Store Packaging Tool" tab) with the same values — this generates the `Package.appxmanifest` XML for reference.

---

## Step 3 — Add Required Store Image Assets

Place the following PNG files in `assets/icons/`:

| File | Size | Usage |
|---|---|---|
| `StoreLogo.png` | 50×50 | Store listing icon |
| `Square44x44Logo.png` | 44×44 | Taskbar + title bar |
| `Square150x150Logo.png` | 150×150 | Start menu medium tile |
| `Wide310x150Logo.png` | 310×150 | Start menu wide tile |
| `Square310x310Logo.png` | 310×310 | Start menu large tile |
| `SplashScreen.png` | 620×300 | App launch splash |
| `icon.ico` | 256×256 | Window icon (ICO format) |
| `icon.png` | 512×512 | High-res source icon |

> **Tip:** Use [GIMP](https://gimp.org), [Figma](https://figma.com), or the free [Store Assets Generator](https://www.pwabuilder.com/imageGenerator) to batch-generate all sizes from a single 512×512 source image.

---

## Step 4 — Build the App

```bash
# Install dependencies (first time only)
npm install

# Build frontend + compile backend
npm run build

# Compile Electron TypeScript
npm run electron:build

# Package as MSIX (outputs to dist-electron/)
npm run electron:pack
```

If the build succeeds, you'll find:

```
dist-electron/
  UWP Weather Forecast-1.0.0-Setup.exe     ← NSIS installer (sideloading)
  UWP Weather Forecast-1.0.0.msix          ← Microsoft Store package
```

---

## Step 5 — Test the MSIX Locally

Before submitting, install the MSIX on your own machine:

```powershell
# Enable Developer Mode first (Settings → Privacy & Security → For developers)
Add-AppxPackage ".\dist-electron\UWP Weather Forecast-1.0.0.msix"
```

Launch the app from the Start menu to verify it works correctly.

To uninstall:
```powershell
Get-AppxPackage *WeatherForecast* | Remove-AppxPackage
```

---

## Step 6 — Submit to Partner Center

1. In Partner Center, open your reserved app → **Start your submission**.
2. Complete the required sections:
   - **Pricing and availability** — Free / Paid / Trial
   - **Properties** — Category: `Weather`, Sub-category: `Weather forecast`
   - **Age ratings** — Run the IARC questionnaire (should rate Everyone)
   - **Store listings** — Add description, screenshots (min 1280×800 or 720×1280), and promotional images
3. In the **Packages** section, upload the `.msix` file from `dist-electron/`.
4. Click **Submit to the Store** and wait for certification (typically 1–3 business days).

---

## Step 7 — Post-Certification

- Partner Center will email you when the app passes certification.
- Your app will appear in the Microsoft Store search within 24 hours of approval.
- To publish an update, increment `version` in `package.json` and `electron-builder.yml`, rebuild, and submit a new package.

## Capabilities Declared

The `Package.appxmanifest` (generated by the in-app Store Packager) declares:

| Capability | Reason |
|---|---|
| `internetClient` | Required to fetch weather data from Open-Meteo and Gemini APIs |
| `location` (optional) | Can be enabled to auto-detect the user's city |

---

## Troubleshooting

| Problem | Solution |
|---|---|
| `Publisher mismatch` error during MSIX install | Ensure the `publisher` CN in `electron-builder.yml` exactly matches your Partner Center identity |
| App hangs on launch | Check that `dist/server.cjs` exists — run `npm run build` first |
| Gemini AI briefing missing | Set `GEMINI_API_KEY` in `.env.local` before building |
| Certification failure: privacy policy missing | Add a privacy policy URL in Partner Center → Properties |
